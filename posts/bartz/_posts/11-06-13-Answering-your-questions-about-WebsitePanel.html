I’m Bradley Bartz, the project lead for WebsitePanel at Outercurve. I’m also a Microsoft program manager working on the Web Platform and Tooling team.

I’d like to help answer some of the questions that have been raised on the project forum. We (Microsoft) remain committed to supporting WebsitePanel and contributing to the project.

We will be officially releasing 1.2.0 in July. This release includes the following features:

Added: Support for managing private Hyper-V cloud solution implemented with System Center Virtual Machine Manager 2008 R2, System Center Operations Manager 2007 and Dynamic Data Center Toolkit v3 (Beta);

Added: Support of Helicon Ape module for securing files & folders on web-site level (contributed by HeliconTech);

Added: Support for Razor files extensions (*.cshtml, *.vbhtml) in File Manager;

Added: "Notify Overused Databases" scheduled task (contributed by yoda-ict);

Added: Display size for MySQL databases on Edit Database page (contributed by yoda-ict);

Updated: Project code base migrated to .NET Framework 4;

http://www.websitepanel.net/releases/1.2.0-beta


WebsitePanel has generated a tremendous amount of interest and is being adapted by several ISVs for their customers. Over time I expect some of this work to be contributed back to the project.

Ryan Jones is a project manager at Microsoft who is helping to coordinate these community contributions.

Pavel Tsurbeleu is the primary developer from Microsoft contributing to this project. He’s an active contributor on the WebsitePanel forums at http://social.msdn.microsoft.com/Forums/en-US/wspsupport/. Please feel free to contact him directly let him know if there are specific issues you’d like to have resolved before it’s released.

In addition to Pavel, Ryan and myself, we have other resources supporting WebsitePanel. We are committed to enhancing the experience of the WebsitePanel community.

As we plan the next release I’d like to learn more about the features that are important to you. If you would like to contribute to the WebsitePanel project, please contact Ryan or myself at the email addresses blow.

Thank you for investing in Website Panel and helping to keep this a vibrant community.
